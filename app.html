PWA First

Install the site now. Build the full app next.

This approach lets the brand move now without waiting on App Store approval. Readers can install the website app, access major study areas quickly, and use the same content system already powering the site.

Future Bible App

Scripture tools are already the foundation.

The Tanakh, New Testament, Apocrypha, chapter readers, word study tools, cross references, and article system give the brand a strong foundation for a full Bible app experience.

Scripture Reader

Quick access to Tanakh, New Testament, Apocrypha, chapter navigation, verse tools, and study panels.

Articles & Teachings

Read structured teachings, research articles, pull quotes, footnotes, and topic-focused study material.

Dietary Laws

Search clean and unclean foods, hidden ingredients, references, and practical modern examples.

Media

Search

Future Study Tools

Planned tools include bookmarks, highlights, local notes, reading history, offline reading, and reading plans.

Build Roadmap

  • Phase 1 Clean up the current app pages and make `/app.html` the main Organization → App destination.
  • Phase 2 Strengthen the PWA install experience with manifest, icons, install prompts, and service worker strategy.
  • Phase 3 Add app-like dashboard features: continue reading, saved tools, bookmarks, local notes, and reading history.
  • Phase 4 Prepare the Bible app experience for later native packaging if the PWA proves strong with users.
